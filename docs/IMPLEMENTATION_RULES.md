Golden Rules

1.
Never rewrite working code without reason.

2.
Never reduce performance.

3.
Always preserve backwards compatibility.

4.
Implement one phase at a time.

5.
Run tests after every phase.

6.
Do not change public APIs unless absolutely necessary.

7.
Prefer refactoring over rewriting.

8.
Document every architectural decision.

9.
Preserve project compatibility.

10.
Never change the application version.
